Beeswarm

Active IDS made easy

Introduction

Beeswarm is a active IDS project which provides easy configuration, deployment and managment of honeypots and clients. The system operates by luring the hacker into the honeypots by setting up a deception infrastructure where deployed drones communicates with the honeypots and intentionally leaks credentials doing so.

Getting up and running - short version

For this scenario we will presume that 5 linux virtual machines are available, one VM will run beeswarm server and the rest will be used as drones.

Installing and starting the server

On the VM that will be used as server do the following: 

$ pip install beeswarm
Downloading/unpacking beeswarm
...
Successfully installed Beeswarm
Cleaning up...
$ mkdir server_workdir
$ cd server-workdir/
$ beeswarm --server
Installing beeswarm on the drones
$ pip install beeswarm
Downloading/unpacking beeswarm
...
Successfully installed Beeswarm
Cleaning up...
Starting the drones
To be able to initially activate drones, extract a temporary configuration url from the beeswarm managment interface by clicking "Add drone". The configuration url will look something like: https://192.168.1.22/drones/add/ui182uu .
Use the same configuration url to start the beeswarm drones on all the remaining VM's: 
$ mkdir drone_workdir
$ cd drone_workdir
$ beeswarm --config https://192.168.1.22/drones/add/ui182uu

Drone configuration and monitoring
Drones can be configured as honeypots or clients from the management interface.

Getting up and running - long version

Instaling the Beesswarm server is as easy as downloading the package with pip and afterwards starting beeswarm with the --server parameter. Make sure to note the auto generated administrative password:

$ pip install beeswarm

$ mkdir server_workdir
$ cd server-workdir/
$ beeswarm --server
